# CLAUDE.md — src/db/ (SHARP EDGE)

> This folder owns the database schema, types, and migrations.
> Read this before making any changes here.

## What lives here
- `types.ts` — auto-generated TypeScript types (DO NOT hand-edit)
- `seed.ts` — local development seed data
- `schema.sql` — reference schema (informational only)

## Non-negotiable rules
1. NEVER hand-edit `src/db/types.ts` — it is generated by `supabase gen types typescript --local > src/db/types.ts`
2. NEVER hand-edit files in `supabase/migrations/` — use `supabase db diff -f name` to generate
3. ALWAYS run type generation after any schema change — see .claude/skills/db-migration.md
4. EVERY new table must have `semester_id uuid references semesters(id)` — see docs/adr/004-semester-anchor-pattern.md
5. EVERY new table must have RLS enabled — see docs/adr/002-rls-for-role-access.md

## Key tables (quick reference)
| Table | Sharp edge |
|---|---|
| `semesters` | Only one can have `is_active = true` at a time |
| `users` | Role is source of truth — synced to JWT via Auth hook |
| `availability` | Unique constraint on (user_id, session_date_id) |
| `sessions` | Status enum is strict: pending, confirmed, declined only |
| `outreach` | Status enum: prospect, contacted, responded, onboarded |

## After any schema change — run this
```bash
supabase gen types typescript --local > src/db/types.ts
```
Then commit `src/db/types.ts` in the same commit as the migration file.
